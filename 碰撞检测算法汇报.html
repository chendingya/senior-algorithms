算法一

Quadtree

四叉树

分治空间，化整为零

设计策略：分治法

把二维地图递归划分为四个象限，区域内对象超阈值时继续分裂

1

整个2D地图

初始状态，所有对象在一个区域

2

超阈值 → 分裂为4个象限

递归划分空间，直到每个叶节点对象数 ≤ 阈值

3

查询时只搜索局部邻居

稀疏区域的对象不必和其他区域比较

核心思想

让每个对象只和局部邻居打交道

适用场景

✓ 2D静态稀疏

✗ 高动态密集

四叉树

伪代码

// 插入操作
func Insert(node, obj):
    if node.count < MAX:
        node.objects.add(obj)
        return
    if not node.isDivided:
        node.Subdivide()
    for child in children:
        if child.contains(obj):
            Insert(child, obj)

// 范围查询
func Query(node, range):
    if not node.intersects(range):
        return  // 剪枝
    for obj in node.objects:
        if range.contains(obj):
            results.add(obj)
    for child in children:
        Query(child, range)

单次查询O(log n)O(log n)O(n)
全场景O(n log n)O(n log n)O(n²)

最坏情况：所有对象重叠在同一区域 → 树退化为链表

算法二

Sweep and Prune

SAP

降维打击，排序复用

设计策略：降维 + 时间局部性

把二维碰撞简化为一维区间重叠问题，将物体包围盒投影到坐标轴，排序后扫描找重叠

1

投影到X轴

得到N个区间

2

按左端点排序

后续帧用插入排序

3

扫描找重叠

不等式传递性剪枝

关键洞察

帧与帧之间，对象位置变化很小

→ 排序顺序几乎不变
→ 插入排序接近O(n)
→ 时间局部性

不等式传递性

若 A≤B 且 B≤C → A≤C
可安全跳过，大规模剪枝

算法流程与复杂度

SAP

伪代码

func SweepAndPrune(objects):
    Sort(objects, key=aabb.minX)
    // 首次快排，后续插入排序
    candidates = []
    for i = 0 to n-1:
        for j = i+1 to n-1:
            if obj[j].minX > obj[i].maxX:
                break  // 剪枝
            if AABBOverlap(i, j):
                candidates.add(i, j)
    return candidates

复杂度分析

操作 最好 平均 最坏
后续帧排序O(n)O(n)O(n log n)
扫描候选对O(n)O(n+m)O(n²)
全场景O(n)O(n+m)O(n²)

关键：复杂度与输出规模挂钩（output-sensitive）

正确性

正确性与适用边界

SAP

正确性说明

投影保交（分离轴定理）

空间相交 → 任意轴投影必有重叠

逆否命题

轴上不重叠 → 一定不相交 → 安全排除

适合场景

• 对象数量大但碰撞对少
• 帧间运动幅度小
• 对象尺寸相近

不适合场景

• 对象密集且大量重叠
• 瞬间大幅位移
• 尺寸差异极大

SAP的威力在于变化少；变化越大，越接近暴力法

算法三

Gilbert-Johnson-Keerthi

GJK 算法

两体相交，以差判之，决于原点

设计策略：迭代逼近法

GJK不显式构造完整的Minkowski差，而是通过迭代逼近在边界上找关键点

Minkowski 差定义

A ⊖ B = {a - b | a ∈ A, b ∈ B}

关键等价关系

A ∩ B ≠ ∅

⟺ 存在公共点 p ∈ A 且 p ∈ B

⟺ p - p = 0 ∈ A ⊖ B

⟺ 原点 ∈ A ⊖ B

核心转化

两物体是否相交

⟺

形状是否包含原点

算法流程

GJK — 取支、立形、决原

支持点

沿给定方向在Minkowski差上最远的点

单纯形

点 → 线段 → 三角形 → 四面体

三步流程

① 找新支持点
② 若 p·d ≤ 0 → 不相交
③ 加入单纯形，检查是否包围原点

伪代码

func GJK(shapeA, shapeB):
    dir = initialDirection()
    simplex = [Support(A, B, dir)]
    dir = -simplex[0]
    while true:
        p = Support(A, B, dir)
        if dot(p, dir) <= 0:
            return false
        simplex.append(p)
        if ContainsOrigin(simplex):
            return true

func Support(A, B, dir):
    return A.farthest(dir) 
         - B.farthest(-dir)

复杂度分析

GJK

迭代次数有常数上界：2D ≤ 3次，3D ≤ 4次

因素 代价 说明
迭代次数（2D）O(1)最多3次（三角形）
Support（朴素）O(n)遍历所有顶点
Support（预处理）O(log n)凸体预排序后二分
增量模式≈ O(1)上一帧作起点，1-2次收敛

空间：O(1)（仅存单纯形，最多4点）

Montanari 2017：有符号体积子算法，CPU减少10%

正确性

为什么GJK是正确的？

1

等价转化

A ∩ B ≠ ∅ ⟺ 原点 ∈ A ⊖ B

2

收敛性

距离单调不增 + 常数上界 → 必然收敛

3

完备性

true=包含原点 ✓
false=无法超越原点 ✓

每次只取最有利于逼近原点的点——迭代贪心的力量

实例演示

GJK — 两个2D正方形

问题设定

正方形A 中心(0,0) 边长2

(-1,-1), (1,-1), (1,1), (-1,1)

正方形B 中心(1.5,0) 边长2

(0.5,-1), (2.5,-1), (2.5,1), (0.5,1)

→ X方向重叠0.5 → 应判定为相交

执行过程

1

d=(1,0) → Support = (0.5, 1)

2

d=(-0.5,-1) → Support = (-3.5, -2)

3

线段不包围原点 → 取第三个点

4

原点在三角形内 → 返回 true ✓

只用3次Support调用，没有遍历任何物体的全部顶点

延伸方向

从GJK出发

1

EPA

GJK只能判定是否相交

EPA计算穿透多深

应用：物理响应方向与深度

2

连续碰撞检测

离散检测可能穿墙

CCD沿轨迹做扫掠

典型：子弹、高速赛车

3

非凸体处理

GJK只适用于凸体

实际模型多为凹形

需凸分解（V-HACD等）

GJK是起点，不是终点

不是比谁算得准
而是比谁排除得快

先粗筛再精算，用最小代价得到正确答案
